# 数字分身使用手册

## 1. 基础准备
```bash
/Users/zqs/Downloads/project/myself/ai-chief/agent.sh projects-discover --root /Users/zqs/Downloads/project --max-depth 2
/Users/zqs/Downloads/project/myself/ai-chief/agent.sh projects-list --enabled-only
/Users/zqs/Downloads/project/myself/ai-chief/agent.sh ai-tools
```

## 2. 给指定项目下达“真实执行任务”
```bash
/Users/zqs/Downloads/project/myself/ai-chief/agent.sh project-execute-task \
  --project-id digital-ecology \
  --objective "实现登录错误提示优化" \
  --acceptance "错误提示明确；测试通过" \
  --ai-tool codex
```

说明：
- 上述命令会生成执行链请求（通常包含 1 个 AI 编码步骤 + N 个验证步骤）。
- 你会拿到多个 `request_id`。

## 3. 审批并执行执行链
```bash
/Users/zqs/Downloads/project/myself/ai-chief/agent.sh project-approve-chain \
  --request-ids req-xxx,req-yyy,req-zzz \
  --approved-by zqs \
  --reason "approved execution chain" \
  --execute
```

## 4. 单条命令审批（手动模式）
```bash
/Users/zqs/Downloads/project/myself/ai-chief/agent.sh project-command-request \
  --project-id digital-ecology \
  --command "git status" \
  --reason "daily check"

/Users/zqs/Downloads/project/myself/ai-chief/agent.sh command-approve \
  --request-id req-xxx \
  --approved-by zqs \
  --reason "approved" \
  --execute
```

## 5. 项目任务池模式（非即时执行）
```bash
/Users/zqs/Downloads/project/myself/ai-chief/agent.sh project-add-task \
  --project-id digital-ecology \
  --task-type weekly_report \
  --objective "梳理本周进展与阻塞"

/Users/zqs/Downloads/project/myself/ai-chief/agent.sh run-queued --limit 20
```

## 6. 状态与排障
```bash
/Users/zqs/Downloads/project/myself/ai-chief/agent.sh status
python3 /Users/zqs/Downloads/project/myself/ai-chief/runtime/command_guard.py list-requests --status pending
```

## 7. API 模式
```bash
python3 /Users/zqs/Downloads/project/myself/ai-chief/runtime/agent.py serve --host 127.0.0.1 --port 8787
```

常用接口：
- `GET /status`
- `POST /projects/execute-task`
- `POST /commands/approve`
- `POST /projects/discover`
