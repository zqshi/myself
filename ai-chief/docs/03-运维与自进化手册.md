# 运维与自进化手册

## 1. 每日运行建议
1. 扫描新任务并执行：`agent.sh run --auto-activate`
2. 处理待审批请求：`command_guard.py list-requests --status pending`
3. 检查健康状态：`agent.sh status`

## 2. 自进化循环
```bash
/Users/zqs/Downloads/project/myself/ai-chief/agent.sh growth-plan --window 200 --top-k 3
/Users/zqs/Downloads/project/myself/ai-chief/agent.sh growth-request --window 200 --top-k 3 --cwd /Users/zqs/Downloads/project/myself
/Users/zqs/Downloads/project/myself/ai-chief/agent.sh evolve --window 200 --auto-activate
```

## 3. 进化门禁策略
- 关键阈值：`configs/thresholds.yaml`
- 审批日志：`logs/command-audit.jsonl`
- 策略历史：`policy_changes`（SQLite）

## 4. 回滚策略
1. 若指标劣化，停止自动激活。
2. 切换回上一个稳定策略版本。
3. 对可疑技能或命令模板降权/禁用。

## 5. AI 工具模板维护
文件：`configs/ai-tools.json`

建议：
1. 优先维护 `codex` 作为默认工具。
2. 其他工具作为备选，逐步灰度。
3. 修改模板后先在单项目试运行，再放大到全项目。

## 6. 风险控制清单
1. 高风险命令必须审批。
2. 不允许未注册目录执行命令。
3. 不允许跨项目误执行：始终使用 `project_id` 路由。
4. 每次任务执行必须附带验收标准。

## 7. 推荐自动化
1. 每天 09:00 触发 `run`。
2. 每天 18:00 触发 `growth-plan + evolve`。
3. 每小时轮询 `/status` 与 pending 请求数量。
