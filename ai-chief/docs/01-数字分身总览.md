# 数字分身 Agent 总览

## 1. 目标
本系统用于把你从“执行者”升级为“审批者/决策者”：
- 统一管理多个项目（`/Users/zqs/Downloads/project`）
- 通过 AI 工具与本地命令执行实际任务
- 在可审计、可审批、可回滚前提下持续自进化

## 2. 核心能力
1. 多项目管理：自动发现并注册项目，按 `project_id` 路由任务与命令。
2. 真执行流水线：`project-execute-task` 生成“AI编码 + 验证命令”执行链。
3. 命令风控：命令白名单、审批门禁、审计日志。
4. 自进化：基于 `episodes/feedback` 执行 `train-cycle` 和 `growth-plan`。
5. API 化：本地 HTTP 接口可接入其他系统（IM、自动化、Webhook）。

## 3. 目录结构（关键）
- `runtime/agent.py`：统一入口（CLI + API）
- `runtime/command_guard.py`：命令策略执行与审批
- `runtime/execution_pipeline.py`：AI执行链构建
- `runtime/orchestrator.py`：任务编排与策略迭代
- `configs/command-policy.yaml`：命令权限策略
- `configs/projects.json`：项目注册表
- `configs/ai-tools.json`：AI 工具模板配置
- `configs/project-goals.json`：项目目标基线
- `skills/*/SKILL.md`：技能定义
- `memory/tasks.db`：任务与反馈数据

## 4. 运行角色分工
1. 你：定义目标、审批高风险动作、确认策略升级。
2. 数字分身：执行任务、跑验证、沉淀日志、提出能力升级建议。

## 5. 关键原则
1. 不绕过审批：高风险命令必须 `request -> approve -> execute`。
2. 不盲目进化：指标不达标不发布；劣化即回滚。
3. 不做黑盒：所有执行与决策均可追溯。
